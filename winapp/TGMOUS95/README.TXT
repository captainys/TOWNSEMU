Windows 95用マウスインテグレータ TGMOUSE.EXE

津軽上で走るWindows 95でマウスを利用するにはいくつかの方法があります。最も簡単なのは、MX/MAで追加されたハイレゾCRTCを有効にした状態だと、津軽は自動的にマウスカーソルをホストOSのマウスカーソルに追従させます。

しかし、この方法でも制限はあって、アプリケーションがカラーマウスカーソルを使うとき、あるいはペイントブラシで絵を描くときなどカーソルが消えているとき、マウスを動かすことができません。

また、FMT-3631/3632 (Power 9000/9100)を有効にして高解像度モードでWindows 95を使いたい場合などは、この方法は使えません。

あるいは、マウスの中ボタンを押して、Differential Mouse Integrationを有効にする方法もあります。この方法ではぎこちないですが、大体マウスの操作ができます。しかし、ホストOSと津軽間の行き来がスムーズにできなくなります。

この問題を解決するのがこのTGMOUSE.EXEです。

Windows 95起動と同時にTGMOUSE.EXEが起動するようにしておくと、FMT-3631/3632使用時でも起動時からスムーズにマウスを使うことができます。

Windows 95起動と同時にTGMOSUE.EXEを起動させるには、ふたつ方法があります。

A. サービスとして登録してしまう。
この方法が一番確実です。次の手順で操作してください。

(1) TGMOUSE.EXEをWindows 95からアクセスできるドライブにコピーする。
(2) Windows 95上でregedit.exeを起動する。
(3) HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows\CurrentVersion を開く。
(4) その場に、RunServices というキーが無い場合は、作成する。
(5) RunServices キーを開く。
(6) 文字列を新規作成。名前はなんでもいいけど、とりあえず TGMOUSE としておく。
(7) 生成した文字列をダブルクリックして、「値のデータ」に、 "D:\PATH_TO_TGMOUSE\TGMOUSE.EXE" を書き込む。

これで、Windows 95起動と同時にTGMOUSE.EXEが使えるようになります。



B. スタートメニューのスタートアップに登録する。
スタートメニューのスタートアップにTGMOUSE.EXEのショートカットを登録してもOKです。ただし、この方法だと、Windows 95のネットワークコンポーネントをインストールすると、ログイン名とパスワードを聞いてくるようになり、その画面の次に進むまでマウスが使えません。
